The Basics

Here's a data model

Templar.dataModel('Example',{
   SimpleSelect : [1,2,3],

   ComplexSelect : [
     {text : 'uno', value : '1'},
     {text : 'dos', value : '2'},
     {text : 'tres', value : '3'}]
});
The first argument to Templar.dataModel() is the modelName. Each property of the second argument is called a data model attribute (DMA).

Using your data models in templates

To make use of DMAs in your template you use specially formatted variable called a nonterminal (NT). It looks like this: {{modelName,attributeName}} The example above has a ',' where a '.' should be. This is to keep Templar from trying to replace the NT with a valid model attribute (which there isn't because we haven't defined a model named "modelName" with an attribute named "attributeName."

Using Selects

Binding

To bind a DMA to a select, place a valid non-terminal (NT) between a select tag. <select>{{modelName,attributeName}}</select>

Using different types of lists

Notice the difference between the SimpleSelect and ComplexSelect attributes.
Templar will use each element in the list as both the name and value
You can specify different names and values by binding to a list of objects that have name and value properties

The 'current_selection' property

Once a list is bound to a select, it gets a "current_selection" property that can be used to get/set the value of the currently selected option.

Selecting an option using the 'current_selection' property

To set a select's current selection, assign 'current_selection' to a value that is in the list. In the case of a complex list, Templar will recognize both valid text and value properties
